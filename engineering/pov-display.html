Persistence of Vision Display

Electronics Arduino Fabrication Programming

The goal of this project was to build an LED display using persistence of vision — the fact that the human eye retains images for a short period of time after it sees them. This means we can show the eye a bunch of images quickly, and it combines them into one. This was built with a friend and mentor of mine, Steve Choate.

How It Was Made

This was clearly built in a prototype form — the frame put together from wood scraps, a popsicle stick mounted to a DC motor using a random gear that happened to fit the shaft, and hot glue gobbed on everywhere.

On the popsicle stick, there is an Arduino mini controlling everything. The LEDs were scraped off of an LED strip and soldered directly onto a piece of copper circuit board, in order to pack them as close together as possible. Tiny surface mount resistors were soldered onto the board as well. On the underside of the popsicle stick, an optical sensor was attached in order to sense when each full revolution had been completed.

On the base, Steve built a small circuit board to act as a speed controller for the motor. We decided to keep power completely separate from the motor and the other electronics.

LED and circuit board assembly on the popsicle stick

Power Contacts

Getting power to the spinning board was the hardest challenge of this project. Direct wires would just get tangled as it spins, so we needed to create some contacts.

For the positive line, we glued a washer onto the motor and attached a springy piece of bent copper on the popsicle stick. These two metal surfaces stay in contact with each other throughout the duration of the revolution.

Washer contact for positive line

Springy copper contact on motor shaft for ground

We thought about putting a second, larger washer for the ground connection, but that proved to be unnecessary when we realized that the shaft of the motor is completely disconnected from any of the motor's electricity. We just pressed another piece of springy copper against the bottom of the motor shaft. This ended up being a perfect solution, and more reliable than the washer.

Code

The code below allows the user to enter any text in capital letters to be shown on the display. Each letter is stored as a bitmapped column-by-column in the lets[] array, and the code reads out each column at timed intervals synchronized to the rotation.

void setup() {
  s = sizeof(test);
  s = s - 1;
  pinMode(A6, INPUT);
  for (int q = 0; q < 16; q++) {
    pinMode(p[q], OUTPUT);
    digitalWrite(p[q], LOW);
  }
}

void loop() {
  while (digitalRead(18) == HIGH) {
    for (int i = 0; i < s; i++) {
      int y = test[i];
      if (y == 32){ y = 91; }  // 32 is ' ', capital letters A-Z run from 65-90
      y = (y - 65) * 8;
      for (int z = 0; z < 8; z++) {
        for (int x = 0; x < 8; x++) {
          int br = bitRead(lets[y], x);
          if (br == 1) {
            digitalWrite((p[x] + 8), HIGH);
          }
          if (br == 0) {
            digitalWrite((p[x] + 8), LOW);
          }
        }
        y = y + 1;
        delayMicroseconds(150);
      }
      for (int q = 0; q < 16; q++) {
        digitalWrite(p[q], LOW);
      }
    }
  }
}

Next Steps

In a future version of this project, it would be interesting to experiment with displaying text and or patterns so that they appear flat. This would likely first require an updated design — smaller LEDs that go all the way to the center would give needed resolution.
