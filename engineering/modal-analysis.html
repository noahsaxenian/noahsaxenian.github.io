Procedure for Modal Analysis

The process starts by exciting an object (in this case the metal plate) and measuring its response. In this case, I am using a small modal impact hammer to tap the plate, and a laser vibrometer to measure the velocity of the plate over time.

Test Stand

I built the aluminum testing rig shown below to securely mount the laser to a rigid frame. The laser hits an angled mirror which reflects the beam up into the underside of the plate. The plate is balanced on four small pieces of foam suspend it and keep it isolated from the rest of the system. The frame is build from aluminum T-slot extrusions. The mounts for the laser, plate, and mirror bracket are waterjet from 1/4” aluminum, and the mount for the mirror is 3d printed. CAD files here.

The raw data from both the vibrometer and hammer is a time signal. I used the Oberlin Acoustics App to synthesize this data into a frequency response function (FRF). This converts both time signals into the frequency domain using a Fourier Transform, and then compares them to describe the relationship between the input force and the resulting output response.

ObiApp developed by Chris Rogers and Joseph Curtin

Here I implemented a modal analysis technique called Circle Fitting to reconstruct the frequency response functions synthesized above. The goal of this step is to fit a mathematical model of the FRF behavior to the frequency domain data.

Circle Fitting is a single degree of freedom (SDOF) technique, meaning it looks at only one resonance peak at a time. This relies on a major assumption: the peaks are spaced far enough apart to barely impact each other.

Step 1:

Identify approximate locations of each resonant frequency (peaks) on experimental data.

Step 2:

Fit a circle to each peak to estimate resonant frequency, damping factor, and modal constant.

Step 3:

Reconstruct full FRF using coefficients determined above for each peak.

The above process of reconstructing a frequency response function corresponds to a single impact point and a single response point. Tapping the hammer at different locations will produce different FRFs. That is to say, the magnitude and phase of modal constants will be different, but the resonant frequencies and damping coefficients should be the same — those are each properties of the modes, not the locations.

To get a more comprehensive understanding of how the test subject responds, we perform a “roving hammer test” by keeping the laser pointed at one spot and impacting at many points around the plate. In this case, I impacted 16 locations in a grid on the metal plate. By comparing the modal constants of each location — for each mode — we can reconstruct the basic motion of each mode shape.

Mode shapes of the metal plate

Performing this procedure on a wooden plate with a cutout simulates the challenges of the f-holes on a violin, especially when we want to interpolate between measurement points.

Testing on a Violin

With these tools that I developed, I performed modal analysis of a violin top plate, compared mode shapes with existing literature, and evaluate the performance of the analysis tools. Using a roving hammer impact test and laser vibrometer measurements, frequency response functions were collected at 75 locations and processed with a my custom circle-fitting algorithm to extract modal parameters. These parameters were used to reconstruct and visualize mode shapes, from which the CBR, A1, and B1+ signature modes were identified. A full report on this testing is available here.

Thoughts and Reflections

  • The hope is that this process can be applied to the analysis of musical instruments (in particular the violin). Developing a better understanding of how structural changes affect acoustics has the potential to provide valuable insight to assist makers in the construction process.
  • Modal analysis is hard when multiple modes have the same resonant frequencies (peaks on top of each other). My software is not currently capable of handling these scenarios, so care by the user must be taken to interpret its results.
  • Endless room for future development includes synthesizing data across multiple tests, overlapping peak analysis, and a more automated system to reduce the need for user input.
