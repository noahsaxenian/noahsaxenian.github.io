SmartLight

Research Arduino Machine Learning NeoPixel CAD

This is a project I am currently working on as a research assistant. The SmartLight is an offshoot of the SmartMotor — a device that can be trained, instead of programmed, to move in different ways in response to sensory inputs. The SmartMotor is being developed as an open source, low-cost tool for teaching the basics of robotics and artificial intelligence.

The SmartLight is meant to be a variation of the SmartMotor, with visual output instead of physical. With a NeoPixel ring embedded on one of the faces of the cube, it can be trained to output different color, brightness, or number of LEDs showing based on different sensory inputs.

How It Works

On the top of the device there is a potentiometer, a button, an on-off switch, and an LCD display which shows a graph. The x-axis shows sensor input, the y-axis shows output range. On one side, there are two buttons which cycle up and down through different screens. On the other side, there is a port to plug in an array of sensors.

There are 5 screens:

  • Training screen for color
  • Training screen for brightness
  • Training screen for number of LEDs
  • Screen to delete points (X icon)
  • Screen to run data (play icon)

Overview of the 5 screens

For each of the three parameters (color, brightness, number), points can be added on each respective training screen by finding the desired output value with the potentiometer, the desired input state of the sensor, and then pressing the button. My Contribution

The framework of this project was already in place as a SmartMotor. I designed and 3D printed the side panel to house a NeoPixel ring. I kept two layers of 3D printed plastic on top of the LEDs to keep a flush surface when the lights are off, but allow enough light to shine through.

The code from the original SmartMotor only has one training screen (and is only trainable on one parameter, position). I modified the code to display the three different training screens, and to merge the training data from each of the parameters.

Next Steps

I am currently building a set of SmartLights to bring into a classroom and get feedback from students. The interface is in need of work, and the challenge of how best to display training three different parameters has many different potential solutions. Seeing how more people interact with the device and getting their feedback will have the biggest impact on design decisions.
